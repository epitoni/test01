jQuery and HTML5

Summary / あらまし

HTML5, an HTML standard that supports multimedia such as graphics, WebAPI, a mechanism for APIs on the Web, and JavaScript libraries that simplify usage or realizing higher functions are the major sources of rich Web applications such as SPA. In other words, it is the underlying technology for applications that run in a web browser, such as word processing and spreadsheets on Google Drive. In this section, we will introduce various JavaScript libraries developed based on HTML5 and jQuery, which are indispensable for developing rich applications on the Web.

グラフィックス等マルチメディアをサポートしたHTMLの規格 HTML5、 Web上でのAPIの仕組み WebAPI、 それらの操作を簡単にしたり高い機能を実現した各種ライブラリは、 SPA 等のリッチなWebアプリケーションの大きな源流といってよいだろう。 すなわち、Googleドライブ上のワープロや表計算に代表されるような、Webブラウザ上で動作するアプリケーションの基盤技術である。 ここでは、HTML5の基に開発された様々なJavaScriptライブラリを紹介するが、 それらはWeb上のリッチなアプリケーション開発になくてはならないものである。

Explanation / 解説

HTML5

HTML5 is a major update of the HTML standard, and it has brought about a major change in Web content, as it became possible to handle graphics and videos that could not be handled directly in the browser. HTML5 is a standard for HTML itself, a plug-in like Flash is not required. As Steve Jobs's comment on Apple's site on April, 2010, Apple has stopped using Flash on iOS. A detailed description of the process can be found in Steve Would Be Proud: How Apple Won The War Against Flash (2012-06-30). TechCrunch's commentary on a series of processes by Ryan Lawler, "Proud of Jobs: Until Apple Wins the Battle with Flash" (June 2012). Even Adobe, the developer of Flash, announced at their web site (June 29, 2012) that it will stop supporting FlashPlayer on Android 4.1 and later. Adobe finally announced that Flash update and distribution will be terminated by the end of 2020 (July 2017).

HTML5は、HTML規格の大幅な更新であり、これまでブラウザ上で直接は扱えなかったグラフィックスや 動画等が扱えるようになりWebコンテンツに大きな変革をもたらしたといえよう。 HTML5はHTML自体の規格なので、Flashのようなプラグインが必要ない。 ジョブズによるFlashについての見解 (2010年4月)にあるように、AppleはiOSでFlashの採用をやめた。 経緯な詳細は、 TechCrunchによる一連の経緯の解説 Ryan Lawler「ジョブズの誇り：AppleがFlashとの戦いに勝つまで」(2012年6月)にある。 Flashの開発元のAdobeさえも 同社のWebサイトでの発表(2012年6月) で、Android 4.1以降でのFlashPlayerのサポートを中止するとしている。 また、 2020年の年末をもってFlashのアップデート及び配布を終了する(2017年7月) と発表した。

jQuery and JavaScript libraries using jQuery / jQueryとそれを利用したJavaScriptライブラリ

jQuery has been considered the de facto standard library for asynchronous communication and DOM manipulation. In fact, it is still widely used (e.g., The Most Popular JavaScript Libraries in The 40 Best JavaScript Libraries and Frameworks for 2022 ). However, due to the increasing functionality of JavaScript itself etc., some people are concerned about its future ( The future of jQuery, jQuery Mobile is no longer supported, 【2022年5月現在】jQueryはオワコン？勉強の必要性や将来性についてまとめてみた, jQuery UIとjQuery Mobileがついに開発終了, ). Therefore, we have removed jQuery from the contents and sample programs in this issue (2022-06-03).
jQuery is a popular JavaScript library that support Ajax, event processing, dynamic rewriting its own HTML. Many JavaScript libraries (jQuery plug-ins) enhancing jQuery's functions have been developed. It made possible to include rich functions into web contents as stand-alone applications, and enhance client's processing capacity on web browsers.
As shown on the top tabs of the homepage, jQuery's plug-ins are not only as jQuery UI, jQuery Mobile (these are developed by themselves), but also as many in The jQuery Plugin Registry. You can search there using keywords (Now moved to npm ), e.g. plot or chart. There are more and more plug-ins which are not registered. In the next section, let me introduce some plug-ins.

jQueryは、非同期通信やDOM操作等、デファクトスタンダードなライブラリだと目されてきました。 実際現在も大変多く使われています(例えば、 The 40 Best JavaScript Libraries and Frameworks for 2022 で、The Most Popular JavaScript Libraries)。 しかし、JavaScript自体の高機能化等から、将来を危ぶむ声もあります( The future of jQuery, jQuery Mobile is no longer supported, 【2022年5月現在】jQueryはオワコン？勉強の必要性や将来性についてまとめてみた, jQuery UIとjQuery Mobileがついに開発終了, )。 そこで、今回の内容やサンプルプログラムからjQueryを外しました(2022-06-03)。
jQuery は、 Ajax、 イベント処理、動的なHTMLの書き換え等をサポートするJavaScriptライブラリである。 また、その機能を拡張する多くのJavaScriptライブラリ(jQueryプラグイン)が開発されており、 Webアプリケーションのブラウザ(クライアント)側での処理能力を飛躍的に進展させるとともに、 Webコンテンツに関しても今まではアプリケーションでしかできなかったような様々なリッチな機能を入れ込むことが可能になってきた。
jQueryのプラグインは、 jQueryのホームページのトップのタブにあるように、 内部で同時に開発されている jQuery UIや jQuery Mobile 以外にも、 jQueryの公式サイトにある プラグイン登録サイト (現在は npm に移行)に多数が登録されている。 ここでキーワードで検索することができ、例えば、 グラフのプロット「plot (又はchart)」 等で検索することができる。 さらに、このサイトに登録されていないものでも、よく使われているものが多数ある。 次節の「実例」で、HTML5やjQueryを用いたライブラリをいくつか紹介する。

Another important aspects of HTML5 / HTML5の他の重要な側面

Here, HTML5 introduced mainly as application for web contents. There are other things as followings.

今回は、HTML5のコンテンツへの活用としての側面に関して扱った。 しかし、今回扱っていない内容として、

  • HTML5's Geolocation API can get current location of mobile device.
    HTML5の位置情報機能の Geolocation APIを用いると、モバイルデバイス等の現在位置等を取得できる。
  • By using HTML5's Playing Videos in HTML, movies can be played on web browser without any plug-ins or helper applications. You need attentions for supported few video format.
    HTML5の動画再生機能 を用いると、Webブラウザだけで動画の再生が可能になる。ただし、再生可能な動画形式が限られているので 注意が必要である。
  • By using HTML5's camera control functions MediaStream Recording and Media Capture and Streams, Video recording and streaming will be possible.
    HTML5のカメラ制御機能の MediaStream Recording や Media Capture and Streams を用いると、ブラウザからビデオ撮影やストリーミング等が行えるようになる。
  • WebRTC is a APIs for HTML5 and provides browsers and mobile applications with Real-Time Communications capabilities, e.g. TV conference between browsers.
    HTML5のでビデオ/オーディオの通信や、データ通信を行うための規格 WebRTCを用いると、ブラウザ間のテレビ会議等が可能になる。

ライブラリの実例 / The actual Examples of Libraries

JavaScript libraries using HTML5
HTML5を活用したJavaScriptライブラリ

Let me introduce very few example of JavaScript libraries using HTML5

これはほんの一部であるが、HTML5を使ったJavaScriptライブラリを紹介する。

  • jQuery UI :
    jQuery UI realizes various types of widgets (e.g. button, checkbox), interactive functions (e.g. drag-and-drop), various effect (e.g. animation). It improve not only usability of web application, but also possibility for various applications. As a personal opinion, with the spread of mobile devices, I think that jQuery UI is no longer needed, and jQuery Mobile is enough.
    jQuery UI は、ボタンやチェックボックス等の様々な ウィジェット や、ドラッグやリサイズといったマウスによる対話性、アニメーション効果等のエフェクトを与える。 これらは、Webアプリケーションの利便性の向上だけでなく、Webコンテンツの可能性も拡げる。 個人的な意見であるが、モバイルデバイスの普及に伴い、 jQuery UIはもはや必要なく、jQuery Mobileだけでよいのではないかと思う。
  • jQueryMobile :
    jQuery Mobile is a HTML5-based user interface system designed to make responsive web sites and apps that are accessible on all smartphone, tablet and desktop devices. It might be introduces later. One of similar libraries is Bootstrap.
    jQuery Mobileは、HTML5ベースのGUIで、 レスポンシブ なWebサイトやWebアプリを実現するためのものです。 スマホ、タブレット、デスクトップ全てに対応しようとするものです。 後で紹介することになると思います。 同様のライブラリとして、 Bootstrap があります。
  • Bootstrap:
    It supports Responsive web design and is useful for building device-independent web front-end UIs. Often used to migrate from jQueryMobile, but not very compatible (personal opinion).
    レスポンシブウェブデザイン に対応し、デバイスに関わらないWebのフロントエンドのUIの構築に役立つ。 jQueryMobileからの移行に使われることも多いが、あまり互換性はない(個人的な見解)。
    • The following jsTree library depends on jQuery. I have not found a good tree treatment library that does not depend on jQuery. Please let me know if you find one.
    以下のjsTreeはjQueryに依存します。 tree系のライブラリでjQueryに依存しないよいものを発見できませんでした。 見つけたら教えてください。
  • jsTree :
    jsTree is a library for showing tree structure as file, folder etc. As demonstration, it can make various type of trees, and not only showing, but also it works as opening, closing etc.
    jsTreeは、ファイルやフォルダー構造の表示等に使われるツリー表示を行うためのライブラリです。 デモ にあるように、様々なリスト表示が可能である。 ただ表示するだけではなく、本当に開いたり閉じたりできる。
  • depending jQuery / jQuery依存
    DataTables :
    DataTables is a quite rich library for tables, and demo shows that it can make not only beautiful tables, but also changing number of visible rows, sorting by columns, finding (extracting) by keywords, etc. It therefore realize functions only by desktop applications before.
    DataTablesは、表(table)に関する非常に多機能なライブラリで、 デモ にあるように、綺麗な表が作成できるだけではなく、表示行数の変更、カラム毎の並べ替え、検索(抽出)等もできる。 つまり、アプリケーションでしかできなかったようなことが可能になる。
  • Grid.js:
    It is a jQuery-independent, highly functional table creation library that makes it easy to create tables with search, sorting, paging, and more.
    jQueryに依存しない、非常に高機能な表作成ライブラリで、検索、ソート、ページング等を備えた表が簡単に作成できます。
  • depending jQuery / jQuery依存
    Flot Examples :
    Flot is a library for plotting graphs by using HTML5 and jQuery, with high interactivity. As demo of selection and zooming, it is not possible by pasting image files. Just as a desktop application.
    HTML5とjQueryを使ってグラフを描くライブラリ(サンプルページ)。対話性を重視。
    Selection and zooming の例の用に、選択範囲やズームもできます。これはイメージの貼り付けでは絶対にできないことで、 まるでアプリケーションのようである。
  • Chart.js:
    This is also simple library for plotting chart.
    これもグラフをプロットするシンプルなライブラリ。
  • D3.js:
    A powerful library that also handles SVG and excels at data visualization and animation. The examples show its wide range of applications.
    SVGも扱え、データの可視化やアニメーションが得意な強力なライブラリ。 例 を見ると、その応用範囲の広さがわかる。
  • TinyMCE:
    Add WYSIWYG editor functionality on web pages. It is built into many systems, such as Wordpress and Moodle, and shows that the editing was actually done on the browser side.
    Webページ上にWYSIWYGエディターの機能を追加する。 WordpressやMoodle等、多くのシステムに組み込まれていて、実はブラウザ側で編集していたことがわかる。
  • impress.js | presentation tool based on HTML5 & CSS3 :
    A library for making presentation page using HTML5 and CSS3. Graphic of HTML5 can be felt.
    HTML5,CSS3を用いたプレゼンページ作成用ライブラリ(デモページ)。
    HTML5のグラフィック能力の高さが実感できる。
  • HTML5 DEMO | Demo Studio | MDN : Many demos.
    HTML5 DEMO | Demo Studio | MDN 多数あり。
